# BPC Hub 文章发布检查表

复制这份清单到每篇文章的工作记录中。全部确认后再把 `draft` 改成 `false`。

## 原创与事实

- [ ] 文章回答了一个具体问题，不是改写商品列表
- [ ] 没有复制厂家、Amazon 或竞争网站的段落
- [ ] 没有声称“亲测”未实际使用的产品
- [ ] 型号、尺寸、材质、配件、保修等规格已从当前来源核实
- [ ] 医学、功效、安全或监管相关陈述有适当来源与上下文
- [ ] 厂家宣称明确写成“品牌声称”，没有当成独立事实
- [ ] 明确写出适用场景、限制和至少一个关键取舍

## 页面与 SEO

- [ ] `title` 清楚自然，不堆关键词
- [ ] `description` 为 70–170 个英文字符
- [ ] URL 文件名短、全小写、用连字符
- [ ] 标题层级从 H2 开始，结构可以扫读
- [ ] 开头直接回答，结尾有明确总结
- [ ] 站内链接指向至少一个相关栏目、政策页或文章
- [ ] 图片拥有版权或明确授权，并写了有效替代文本

## Affiliate 合规

- [ ] 只有自己的 Associates/SiteStripe 链接
- [ ] 有联盟链接时设置 `affiliate: true`
- [ ] Amazon 链接使用 `amazon-link` 或 `product-card` 短代码
- [ ] 商业链接旁显示 “Paid link”
- [ ] 正文没有写死会变化的价格、折扣或库存
- [ ] 没有承诺通过链接购买一定能获得某项优惠
- [ ] 没有手工复制或下载无授权的 Amazon 商品图

## 健康与隐私

- [ ] 涉及健康内容时设置 `medical: true`
- [ ] 不做个体诊断、治疗建议或结果保证
- [ ] 不收集不必要的读者敏感信息
- [ ] 如新增表单、分析、广告或嵌入内容，已同步更新隐私政策

## 上线验收

- [ ] 本地预览无明显排版问题
- [ ] `python scripts/check_content.py` 无 `ERROR`
- [ ] GitHub Actions 构建为绿色
- [ ] 线上手机宽度可读、导航和链接可用
- [ ] 抽查所有外链，没有错链或错误 tracking ID
- [ ] 记录事实核查日期和需要回访的内容
